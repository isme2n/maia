## Notes

- This matrix locks the Part 4 first-run UX and the relative positioning of support surfaces. It does not replace the detailed Part 3 save/restore wording source of truth in [portable-state-public-contract-matrix.md](/home/asle/maia/docs/contracts/portable-state-public-contract-matrix.md).
- `doctor` is intentionally narrower than `setup`: `doctor` answers shared-infra readiness, while `setup` performs shared-infra bootstrap and then hands off to the agent-specific flow.
- `inspect` and Keryx visibility remain public and supported, but any wording that makes them look like the default bootstrap path is drift.
